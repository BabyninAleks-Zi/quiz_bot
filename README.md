# Quiz Bot

Quiz Bot задаёт вопросы викторины в Telegram и ВКонтакте, проверяет ответы пользователей и хранит вопросы, состояние пользователей, счёт и жалобы в Redis.

## Демо

- Telegram: [@Incredible_space_bot](https://t.me/Incredible_space_bot)
- ВКонтакте: [vk.com/club235528518](https://vk.com/club235528518)

## Возможности

- Telegram-бот с кнопками.
- VK-бот с кнопками.
- Кнопка `Новый вопрос`.
- Кнопка `Сдаться`.
- Кнопка `Мой счёт`.
- Кнопка `Вопрос составлен неверно` после сдачи.
- Проверка ответа пользователя.
- Хранение вопросов и состояния пользователей в Redis.
- Поддержка личных и групповых чатов VK через `peer_id`.
- Парсинг вопросов из `.txt` файлов в кодировке `KOI8-R`.

## Установка

Создайте виртуальное окружение и установите зависимости:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## Redis

Для локальной разработки можно использовать Redis, установленный через Homebrew:

```bash
brew services start redis
redis-cli ping
```

Ожидаемый ответ:

```text
PONG
```

По умолчанию бот подключается к Redis так:

```env
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=
```

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
TG_TOKEN=telegram_bot_token

VK_TOKEN=vk_group_token
VK_GROUP_ID=123456789

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=
```

`TG_TOKEN` нужен для Telegram-бота.

`VK_TOKEN` и `VK_GROUP_ID` нужны для VK-бота сообщества.

`REDIS_DB` можно оставить `0`, если вы используете Redis только для этого проекта.

`REDIS_PASSWORD` можно оставить пустым при локальном Redis без пароля.

## Входные данные для парсера

Файлы с вопросами нужно положить в:

```text
docs/quiz-questions/
```

Файлы должны быть `.txt` в кодировке `KOI8-R`.

Минимальный ожидаемый формат:

```text
Чемпионат:
Название чемпионата

Дата:
00-000-2000

Тур:
Тур 1

Вопрос 1:
[Ведущему: это служебная подсказка, бот её скроет.]
Текст вопроса.

Ответ:
Правильный ответ. Пояснение после точки пользователю угадывать не нужно.

Автор:
Имя автора
```

Важные правила формата:

- блоки разделяются пустой строкой;
- блок вопроса начинается с `Вопрос N:`;
- следующий за ним блок ответа начинается с `Ответ:`;
- служебные подсказки в начале вопроса в квадратных скобках удаляются;
- ответ для проверки упрощается: берётся часть до первой точки или открывающей скобки.

Примеры упрощения ответа:

```text
Батарея (от battre).       -> Батарея
"... сразу слышно".        -> сразу слышно
Говорит. Подробный комментарий -> Говорит
```

## Как Redis хранит данные

Вопросы сохраняются в Redis как JSON:

```text
question_1 -> {"question": "Текст вопроса", "answer": "Ответ"}
question_2 -> {"question": "Текст вопроса", "answer": "Ответ"}
```

Список id всех вопросов хранится отдельно:

```text
question_ids -> ["question_1", "question_2", ...]
```

Состояние пользователя хранится как JSON:

```text
user_tg_<chat_id> -> {
  "last_asked_question": "question_1",
  "last_surrendered_question": "question_1",
  "correct_answers": 2,
  "wrong_answers": 5
}

user_vk_<peer_id> -> {
  "last_asked_question": "question_2",
  "last_surrendered_question": null,
  "correct_answers": 1,
  "wrong_answers": 3
}
```

Жалобы на вопросы хранятся в Redis hash:

```text
question_reports -> question_1: 3
```

## Как работает проверка ответа

1. Пользователь нажимает `Новый вопрос`.
2. Бот выбирает случайный `question_id` из Redis.
3. Бот достаёт текст вопроса из Redis.
4. Бот сохраняет `last_asked_question` в состоянии пользователя.
5. Пользователь пишет ответ.
6. Бот достаёт текущий `question_id` из состояния пользователя.
7. Бот находит правильный ответ в Redis.
8. Бот сравнивает ответ пользователя с короткой версией правильного ответа.
9. Бот обновляет счёт пользователя.

Если пользователь нажимает `Сдаться`, бот сохраняет вопрос в `last_surrendered_question`. После этого можно нажать `Вопрос составлен неверно`, и бот увеличит счётчик жалоб на этот вопрос.

Ключи пользователей имеют формат:

```text
user_tg_<chat_id>
user_vk_<peer_id>
```

`peer_id` в VK используется, чтобы бот работал одинаково в личных сообщениях и групповых чатах.
