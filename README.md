# Quiz Bot

Quiz Bot задаёт вопросы викторины в Telegram и ВКонтакте, проверяет ответы пользователей и хранит текущий вопрос пользователя в Redis.

## Демо

- Telegram: [@Incredible_space_bot](https://t.me/Incredible_space_bot)
- ВКонтакте: [vk.com/club235528518](https://vk.com/club235528518)

## Возможности

- Telegram-бот с кнопками.
- VK-бот с кнопками.
- Кнопка `Новый вопрос`.
- Кнопка `Сдаться`.
- Проверка ответа пользователя.
- Хранение текущего вопроса пользователя в Redis.
- Парсинг вопросов из `.txt` файлов в кодировке `KOI8-R`.

## Структура проекта

```text
quiz_bot/

├── vk_bot.py            # VK-бот

├── README.md
```

Папка `docs/` используется только локально и находится в `.gitignore`.

## Установка

Создайте виртуальное окружение и установите зависимости:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## Redis

Для локальной разработки можно использовать Redis, установленный через Homebrew:

```bash
brew services start redis
redis-cli ping
```

Ожидаемый ответ:

```text
PONG
```

По умолчанию бот подключается к Redis так:

```env
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=
```

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
TG_TOKEN=telegram_bot_token

VK_TOKEN=vk_group_token
VK_GROUP_ID=123456789

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=
```

`TG_TOKEN` нужен для Telegram-бота.

`VK_TOKEN` и `VK_GROUP_ID` нужны для VK-бота сообщества.

`REDIS_DB` можно оставить `0`, если вы используете Redis только для этого проекта.

`REDIS_PASSWORD` можно оставить пустым при локальном Redis без пароля.

## Входные данные для парсера

Файлы с вопросами нужно положить в:

```text
docs/quiz-questions/
```

Файлы должны быть `.txt` в кодировке `KOI8-R`.

Минимальный ожидаемый формат:

```text
Чемпионат:
Название чемпионата

Дата:
00-000-2000

Тур:
Тур 1

Вопрос 1:
[Ведущему: это служебная подсказка, бот её скроет.]
Текст вопроса.

Ответ:
Правильный ответ. Пояснение после точки пользователю угадывать не нужно.

Автор:
Имя автора
```

Важные правила формата:

- блоки разделяются пустой строкой;
- блок вопроса начинается с `Вопрос N:`;
- следующий за ним блок ответа начинается с `Ответ:`;
- служебные подсказки в начале вопроса в квадратных скобках удаляются;
- ответ для проверки упрощается: берётся часть до первой точки или открывающей скобки.

Примеры упрощения ответа:

```text
Батарея (от battre).       -> Батарея
"... сразу слышно".        -> сразу слышно
Говорит. Подробный комментарий -> Говорит
```


## Как работает проверка ответа

1. Пользователь нажимает `Новый вопрос`.
2. Бот выбирает случайный вопрос.
3. Бот сохраняет текущий вопрос чата в Redis.
4. Пользователь пишет ответ.
5. Бот достаёт текущий вопрос из Redis.
6. Бот находит правильный ответ в словаре вопросов.
7. Бот сравнивает ответ пользователя с короткой версией правильного ответа.

Ключи Redis имеют формат:

```text
tg-<chat_id>-question
vk-<peer_id>-question
```
